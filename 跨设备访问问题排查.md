# 🔧 跨设备访问问题排查指南

## 问题现象
从另一台电脑通过局域网IP（如 http://172.19.28.205:5000）访问时：
- ✅ 页面可以正常打开
- ❌ 生成结果后看不到音乐特征数值
- ❌ 雷达图不显示
- ❌ 原始特征数据为空

## 根本原因
应用依赖外部CDN资源，在内网或有网络限制的环境中可能无法加载：

### 1. Chart.js 库（必需）
- **用途**: 绘制雷达图
- **CDN地址**: `https://cdn.jsdelivr.net/npm/chart.js@4.4.0`
- **影响**: 如果无法加载，雷达图将不显示

### 2. Google Fonts（可选）
- **用途**: Inter字体美化界面
- **CDN地址**: `https://fonts.googleapis.com`
- **影响**: 如果无法加载，使用系统默认字体，不影响功能

## 诊断步骤

### 方法1: 使用网络检测页面
1. 访问 `http://172.19.28.205:5000/network-test`
2. 查看各项资源的加载状态
3. 根据检测结果进行针对性处理

### 方法2: 浏览器开发者工具
1. 按 `F12` 打开开发者工具
2. 切换到 **Console** 标签页
3. 查找红色错误信息：
   ```
   Failed to load resource: net::ERR_NAME_NOT_RESOLVED
   或
   Access to XMLHttpRequest ... has been blocked by CORS
   ```
4. 切换到 **Network** 标签页
5. 刷新页面，查看哪些资源加载失败（红色）

## 解决方案

### 方案A: 配置网络访问（推荐）
适用于有网络管理权限的情况：

1. **允许访问CDN域名**
   - `cdn.jsdelivr.net`
   - `fonts.googleapis.com`
   - `fonts.gstatic.com`

2. **配置DNS**
   确保客户端可以解析以上域名

3. **检查防火墙**
   允许HTTPS (443端口) 出站流量

### 方案B: 使用代理或VPN
在访问的电脑上：
1. 配置系统代理或浏览器代理
2. 确保代理可以访问国际CDN
3. 刷新页面重新加载资源

### 方案C: 本地部署CDN资源（开发中）
⚠️ 此功能正在开发中，将在下个版本提供

下载Chart.js到本地：
```bash
# 在项目目录执行
curl -o static/chart.min.js https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js
```

然后修改 `templates/index.html` 中的引用。

## 验证修复

修复后，重新访问并检查：

### ✅ 正常状态
- 雷达图显示音乐特征（可舞性、能量、情感价等）
- 特征列表显示13个音乐属性及数值
- 原始特征数据显示完整的JSON格式
- 浏览器控制台无红色错误

### ❌ 仍有问题
1. 重新执行诊断步骤
2. 检查服务器日志
3. 确认后端API是否正常返回数据：
   ```bash
   curl http://172.19.28.205:5000/api/styles
   ```
